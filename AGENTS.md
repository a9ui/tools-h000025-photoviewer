# Project Router

Read in order:

1. `PROJECT.md`
2. `DESIGN.md`
3. `project.toml`

Then inspect GitHub milestone, issues, PRs, Actions, SQLite jobs, and local
`git status`.

Use Codex for small edits and control work. Use Cursor for normal
implementation through a GitHub branch and PR. Use PRO/ChatGPT Project for
milestone-level judgment.

Linear is not used.
