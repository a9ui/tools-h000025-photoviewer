# Local Native M15 - Date Section Parity

## Goal

Advance one small native date-section/date-filter parity row after M14 without
redesigning the UI, touching the browser app, using Linear, deploying, touching
H000033, or starting enhancement workers.

## Canonical Intent

Use `docs/local-native/native-intent-source.md` as the product-intent source.
The target remains the existing PhotoViewer workflows ported into fast local
native software. M15 does not invent a new gallery design or visual direction.

## GitHub Route

- Milestone: #20 `Local Native M15 - Date Section Parity`
- Issue: #85 `M15-001 Map native date section parity`
- Branch: `codex/local-native-m15-date-section-parity`

Linear is intentionally not used.

## First PR Scope

Allowed:

- `local-native/**`
- `scripts/start-local-native.ps1`
- `docs/local-native/**`
- `tasks/local-native-m5/browser-regression-matrix.md`
- `tasks/local-native-m15/**`

Implemented in this first slice:

- native `Date` preset selector for `All dates`, `Today`, `7d`, `30d`, and
  `This year`;
- browser behavior mapping from Sidebar `dateFrom` / `dateTo` presets and
  search filtering by image `createdAt`;
- native filtering by `NativeImageRecord.CreatedAtUtc` as a local calendar
  date;
- composition with existing native search, favorite, folder, and sort controls;
- persisted native SQLite `date_filter` setting;
- `-HeadlessDateFilterSmoke` proving the date presets on a relative-date
  fixture without browser runtime, local HTTP server, Node, or passive
  enhancement changes.

Still deferred:

- date section header layout/grouping in native list/grid;
- manual date range inputs;
- folder range selection pending replacement/custom folder bucket control;
- drag/open parity, placeholder behavior, and native thumbnail warmup UI;
- preview tabs and pin/unpin/restore behavior;
- bulk recycle confirmation and destructive delete sweeps;
- explicit enhancement queue UI and enhanced-only filter;
- full browser API/error equivalence.

Forbidden unless explicitly approved:

- `src/**`
- H000033
- deployment
- automatic enhancement workers
- deletion of `.cache/thumbs`, `.cache/display`, `.cache/enhance`,
  favorites, albums, settings, or native SQLite state
- Chrome profile reads or browser-state scraping
- new UI concepts that do not map back to the existing PhotoViewer behavior
